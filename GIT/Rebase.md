# Rebase

- Rebase를 하든지, Merge를 하든지 최종 결과물은 같고 커밋 히스토리만 다르다는 것이 중요

+ merge는 예를 들어 부모커밋이 같은 두 브랜치에 커밋두개를 (C2,C3) 이라고 한다면   
master브랜치인(C2) 뒤에 새로운 커밋 C4를 새롭게 만든다.
+ rebase는 위와 조건이 같은 상황이다 생각한다면    
C2에서 변경된 사항을 Patch로 만들고 이를 다시 C3에 적용시키는 방법이다.
    + 두 브랜치가 나뉘기 전인 공통커밋(부모커밋)으로 이동하고 나서 그 커밋부터 지금 Checkout 한 브랜치가 가리키는 커밋까지 diff를 차례로 만들어 어딘가에 임시로 저장해 놓는다. Rebase 할 브랜치(C3의 브랜치)가 합칠 브랜치(C2의 브랜치, master)가 가리키는 커밋을 가리키게 하고 아까 저장해 놓았던 변경사항을 차례대로 적용한다.
